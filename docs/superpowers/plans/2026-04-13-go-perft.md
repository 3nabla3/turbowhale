# go perft Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement `go perft <depth>` UCI subcommand that prints per-move node counts (perft divide) for debugging move generation correctness.

**Architecture:** A new `src/perft.rs` module exposes `perft(position, depth) -> u64` (recursive leaf counter) and `perft_divide(position, depth) -> Vec<(Move, u64)>` (per-root-move breakdown). `GoParameters` gains a `perft_depth: Option<u32>` field; the UCI handler detects it and runs perft on the main thread instead of spawning a search thread, printing divide output then returning without a `bestmove` line.

**Tech Stack:** Rust, existing `board::{Position, Move, apply_move, from_fen}`, existing `movegen::generate_legal_moves`, existing `uci::{move_to_uci_string, parse_go_parameters, GoParameters}`

```rust
    if depth == 0 {
```

```rust
pub fn perft_divide(position: &crate::board::Position, depth: u32) -> Vec<(Move, u64)> {
    if depth == 0 {
        return Vec::new();
    }
    generate_legal_moves(position)
        .into_iter()
        .map(|chess_move| {
            let node_count = perft(&apply_move(position, chess_move), depth - 1);
            (chess_move, node_count)
        })
        .collect()
}
```

```rust
    if let Some(depth) = parameters.perft_depth {
        let divide = perft_divide(&state.current_position, depth);
        let total: u64 = divide.iter().map(|(_, count)| count).sum();
        for (chess_move, count) in divide {
            writeln!(output, "{}: {}", move_to_uci_string(chess_move), count).unwrap();
        }
        writeln!(output, "").unwrap();
        writeln!(output, "Nodes searched: {}", total).unwrap();
        output.flush().unwrap();
```
